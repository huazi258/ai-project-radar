# 页面结构设计

## 1. 页面清单

第一版页面只需要支撑核心闭环：

登录 → 新建记录 → 查看记录 → AI 分析 → 生成项目卡片 → 生成 PRD → 复制 Markdown

MVP 必做页面：

```text

/
首页
首页用于提供产品名称和开始使用入口。
模块：
产品名称
一句话介绍
开始使用按钮

/login
登录页
模块：
产品 Logo 或名称
邮箱输入框
登录按钮
登录状态提示
第一版可以只做邮箱登录。

/dashboard
控制台首页
第一版可延后。
模块：
欢迎语
本周记录数量
项目数量
最近记录列表
最近项目列表
快速新建记录按钮

/records
记录列表页
模块：
页面标题
新建记录按钮
记录卡片列表
每条记录展示：
标题
类型
标签
创建时间

/records/new
新建记录页
模块：
标题输入框
类型选择
标签输入
正文内容输入框
保存按钮

/records/[id]
记录详情页
模块：

记录标题
记录正文
标签
创建时间
AI 分析按钮
AI 分析结果展示区
生成项目卡片按钮
复制 Markdown 按钮

/projects
项目列表页
模块：

页面标题
项目卡片列表

每张项目卡片展示：

项目名称
项目简介
项目状态
技术栈标签
创建时间

/projects/[id]
项目详情页
模块：

项目名称
项目简介
目标用户
核心痛点
MVP 功能
技术栈
生成 PRD 按钮
PRD Markdown 展示区
复制 Markdown 按钮

/settings
设置页
第一版可延后。
第一版只做简单展示。
模块：
用户信息
当前版本说明
退出登录按钮
```
