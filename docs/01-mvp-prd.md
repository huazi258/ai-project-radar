# AI Project Radar MVP PRD

## 1. MVP 目标

第一版目标是完成一个可以本地运行、可以保存数据、可以调用 AI 分析记录、可以生成项目卡片和 PRD 的全栈 MVP。

重点不是功能多，而是完整跑通产品闭环。

## 2. MVP 功能清单

MVP 只围绕一条主流程展开：

新建记录 → AI 分析 → 保存分析结果 → 生成项目卡片 → 生成简易 PRD → 复制 Markdown

第一版不追求完整工作台，只保证这条闭环稳定可用。

### 2.1 用户系统

- 用户可以登录
- 用户只能看到自己的记录和项目

### 2.2 记录模块

- 用户可以新建学习记录
- 用户可以查看记录列表
- 用户可以查看记录详情

记录内容包括：

- 标题
- 正文内容
- 记录类型
- 标签
- 创建时间

### 2.3 AI 分析模块

用户可以在记录详情页点击“AI 分析”。

AI 需要输出：

- 内容摘要
- 涉及技能
- 暴露问题
- 下一步行动
- Markdown 总结

如果 AI 分析失败，第一版只需要显示失败提示，并允许用户重新点击分析。
失败结果不需要保存到数据库。

### 2.4 项目卡片模块

用户可以根据一条记录和 AI 分析结果生成项目卡片。

项目卡片包括：

- 项目名称
- 项目简介
- 目标用户
- 核心痛点
- MVP 功能
- 推荐技术栈
- 项目状态

第一版默认每次从最新一条 AI 分析结果生成项目卡片。
项目状态可以默认使用 idea，不需要做复杂状态流转。

### 2.5 PRD 生成模块

用户可以根据项目卡片生成简易 PRD。

PRD 包括：

- 项目背景
- 目标用户
- 核心痛点
- MVP 功能
- 页面结构
- 数据结构
- 验收标准
- 后续迭代方向

### 2.6 Markdown 复制

用户可以复制：

- AI 分析结果 Markdown
- 项目卡片 Markdown
- PRD Markdown

Markdown 只需要支持一键复制，不需要做复杂编辑器。

## 3. 第一版不做的功能

- 不做复杂仪表盘统计
- 不做复杂编辑器
- 不做文件上传
- 不做多人协作
- 不做项目公开展示页
- 不做支付
- 不做管理员后台
- 不做自动爬虫
- 不做移动端适配
- 不做记录编辑和删除
- 不做项目状态流转
- 不做 AI 分析历史管理

## 4. 验收标准

完成 MVP 后，用户应该可以：

1. 登录系统
2. 新建一条学习记录
3. 在记录详情页查看内容
4. 点击 AI 分析
5. 看到结构化分析结果
6. 保存分析结果
7. 生成项目卡片
8. 查看项目详情
9. 生成简易 PRD
10. 复制 Markdown 内容
