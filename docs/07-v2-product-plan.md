# AI Project Radar V2 产品规划

## 1. 第二版定位

第二版将 AI Project Radar 从“单一记录转项目 PRD 工具”升级为“个人 AI 学习工作台”。

第一版解决的是一条完整闭环：

```text
学习记录 -> AI 分析 -> 项目卡片 -> PRD -> Markdown
```

第二版要解决的是用户每天学习、表达和项目思考时的三类高频需求：

```text
学习记录沉淀
结构化表达整理
项目想法孵化
```

第二版的重点不是增加很多复杂功能，而是把第一版已经验证过的 AI 分析能力拆成三个清晰入口，让用户可以更自然地把日常学习过程沉淀下来。

## 2. 目标用户

第二版继续面向个人学习者，尤其是：

- 正在学习 AI、编程、全栈开发的人
- 想把每日学习内容变成可复盘资料的人
- 想提升技术表达和需求表达能力的人
- 想把零散想法逐步整理成项目方案的人

## 3. 第二版核心模块

### 3.1 学习记录

学习记录模块用于记录今日学习情况、具体学习内容、教程笔记、踩坑过程或 AI 工具使用经验。

用户输入原始学习内容后，AI 帮助输出：

- 学习摘要
- 涉及技能
- 当前问题
- 下一步行动
- 适合保存到知识库的 Markdown

这个模块继承第一版记录分析能力，是第二版的基础模块。

### 3.2 结构化表达

结构化表达模块用于把用户输入的一段松散描述，整理成更清楚、更有层次、更适合复制的表达。

适用场景：

- 把口语化想法整理成正式说明
- 把混乱需求整理成条理清晰的任务描述
- 把学习复盘整理成结构化总结
- 把给 Codex 或 ChatGPT 的任务描述整理成更好的提示词

AI 输出方向：

- 优化后的表达
- 要点拆分
- 可复制 Markdown
- 可选的 Codex 任务提示词版本

这个模块不直接生成项目卡片，也不直接生成 PRD。它的边界是“表达整理”。

### 3.3 项目思考

项目思考模块用于把用户输入的项目想法、当前需求、学习灵感或痛点，整理成项目方向。

AI 帮助输出：

- 项目方向
- 目标用户
- 核心痛点
- MVP 范围
- 推荐技术方案
- 风险和后续迭代
- 简易 PRD

这个模块继承第一版项目卡片和 PRD 生成能力，但入口更明确：用户可以直接输入项目想法，不一定必须先从学习记录生成。

## 4. 模块边界

| 模块 | 输入 | 主要输出 | 不负责 |
|---|---|---|---|
| 学习记录 | 学习过程、教程笔记、AI 使用记录 | 学习摘要、技能、问题、行动、Markdown | 不直接做复杂项目规划 |
| 结构化表达 | 松散描述、口语化想法、任务描述 | 清晰表达、要点拆分、可复制 Markdown | 不保存为项目，不生成 PRD |
| 项目思考 | 项目想法、需求、痛点、功能构想 | 项目卡片、MVP、技术方案、PRD | 不做项目管理和任务排期 |

## 5. 第二版核心闭环

第二版不再只有一条线性流程，而是三个入口共享一个个人知识沉淀目标：

```text
学习记录 -> AI 学习分析 -> Markdown 沉淀
结构化表达 -> AI 表达整理 -> Markdown 复制
项目思考 -> AI 项目规划 -> 项目卡片 / PRD
```

其中“项目思考”可以继续连接第一版的 projects 和 PRD 能力。

## 6. 第二版不做什么

第二版暂时不做：

- 多人协作
- 付费系统
- 浏览器插件
- 移动端 App
- 向量数据库和语义搜索
- 自动抓取全网内容
- 完整任务管理系统
- 复杂富文本编辑器
- 公开作品展示页

## 7. 验收标准

第二版完成后，用户应该可以：

1. 从工作台选择学习记录、结构化表达或项目思考。
2. 在学习记录模块记录学习内容并获得 AI 分析建议。
3. 在结构化表达模块输入松散描述并获得清晰可复制的表达。
4. 在项目思考模块输入项目想法并生成项目方向、MVP 和 PRD。
5. 将三个模块的结果复制为 Markdown。
6. 登录用户只能访问自己的数据。
